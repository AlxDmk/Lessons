Игра ""Крести - Нолики"
***********************************
Основная стратегия:
	1. В зависимости от заданного количества необходимых заполненных клеток для одержания победы (NUM) . Расчитывается количесто квадратов NUM x NUN, которые можно построить на игровом поле.
		Например, на поле 5х5 можно построить четыре поля NUM 4х4 (по горизонтале : два с шагом одна клетка и по вертикале с шагом одна клетка) (SIZE_X - num = количество шагов, т.е. два полноценных квадрата). соответственно поле NUM 3х3 можно вписать 9 раз в игровое
		поле 5х5.
		Основной алгоритм: создать массив всех возможных  двумерных массивов NUM x NUM и методом передора проверять:

		1.1. Есть ли победная комбинация после хода : перебираются все возможные двумерные массивы и в случае наличия победной комбинации в каком либо из них игра заканчивается победой ходящего
	
	2. Компьютер
		2.1. пытается  победить перебором массива основного поля на наличие у него комбинации из NUM-1. Если находи, то ставит и побеждает
	    2.2. если не находи своей победной комбинации, то мешает победить игроку перебором массива основного поля на наличие у игрока комбинации из NUM-1. Если находи, то повторно перебирает эту итерацию и вставляет свой DOT. 
		2.3. Если не находит, то ходит рандомно.

	Описание классов:
	class Coss - основной класс с точкой входа + некоторые необходимые методы
	class AllCombo - формирует массив из всех возможных для рассмотрения массивов NUM x NUM
	class Check - "сборщик" всех методов, чтобы не захломлять код созданием новых оюъектов классов. класс Cross обращается только к нему 
	class CheckWin - дочерний класс AllCombo, содержит метод поиска выйгрышной комбинации
	class Disturb -  содержит метод поиска предвыйгрышных комбинаций. К нему ссылается компьютер пытаясь выйграть или помешать играку выйграть (Метод поиска комбинаций по диагонале не реализован из-за нехватки времени: только по вертикале и горизонтале)
